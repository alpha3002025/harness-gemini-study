# Markdown 안에 Markdown 포함하려 할때

## Q1. Markdown 안에서 또 다른 Markdown 코드 블록을 예시로 보여주고 싶을 때 어떻게 하나요?

**A1.** 코드 블록을 감싸는 **백틱(`)의 개수를 조절**하면 됩니다.

보통 코드 블록을 만들 때 3개(```)를 사용하지만, 그 **내용물(코드 블록 예시)** 안에 이미 3개짜리 백틱이 들어있다면, **바깥쪽은 4개(````) 이상**으로 감싸야 합니다.

---

## Q2. 왜 단순히 3개짜리 백틱을 두 번 쓰면 안 되나요?

**A2. Markdown 파서가 "어디가 끝인지" 헷갈리기 때문입니다.**

만약 3개로 시작해서 그 안에 또 3개짜리 블록을 넣으면, 파서는 **안쪽 블록이 끝나는 지점(```)을 전체 블록의 끝**으로 인식합니다. 그러면 나머지 뒷부분의 텍스트들이 코드 블록 밖으로 삐져나오거나 문서 구조가 깨지게 됩니다.

---

## Q3. 4개 말고 5개나 6개를 써도 되나요?

**A3. 네, 가능합니다.**

원칙은 하나입니다: **"가장 바깥쪽 백틱의 개수는 안쪽의 어떤 백틱 덩어리보다도 최소 1개 이상 많아야 한다."**

만약 4개짜리 백틱 블록 자체를 예시로 보여주고 싶다면, 바깥쪽을 5개로 감싸면 됩니다.

---

## Q4. 실제 예시를 보여주세요.

**A4. 아래는 4개짜리 백틱으로 감싸서 내부의 3개짜리 블록을 안전하게 보여주는 예시입니다.**

````markdown
# Step 1: 설정

## 실행 커맨드
```bash
npm run build
npm test
```
````

## Q5. 이 기능은 어디서나 작동하나요?

**A5.** 네, GitHub, VS Code, Obsidian 등 **CommonMark**나 **Gfm**(GitHub Flavored Markdown) 표준을 따르는 거의 모든 현대적인 Markdown 편집기에서 지원하는 정석적인 방법입니다.
